## Known blocker for 1.0.0

Two pre-existing Fix backend test failures (present in the Inc.0 baseline before
any change, unchanged by this stabilization):

- `audisor_backend/tests/fix/test_fix_host.py::test_H1_custom_igniter_calls_package_from_context`
- `audisor_backend/tests/fix/test_fix_host.py::test_H1_package_construction_failure_persists_without_continuing`

Both assert the Fix controller must halt after a `package_from_context`
construction failure; the controller currently continues. This is a genuine
logic discrepancy (test expectation vs controller behavior) that requires a
human decision on intended semantics — it was not silently changed during
stabilization. **CI `ci.yml` will report the Fix-backend job as failing until
this is resolved; 1.0.0 promotion is gated on it.**

## Platform gates not verifiable locally

- **Images anonymously pullable:** requires publishing 0.9.0 images to GHCR
  (the `release-verify.yml` workflow does this on a `v*` tag). Not performed
  locally.
- **All four suites green in CI:** requires a GitHub Actions run; blocked by the
  2 Fix backend failures above.

## Conclusion

0.9.0 is a verified release candidate for the image-packaging and per-product
validators above. Promotion to 1.0.0 remains gated on: (1) resolving the 2
pre-existing Fix backend failures, (2) demonstrating the containerized
end-to-end Fix stack, (3) publishing anonymously-pullable 0.9.0 images, and
(4) a green CI run.
